## App Structure
- Auth routes: (auth)/login, (auth)/signup, (auth)/onboarding
- Tab routes: directory, promotions, community, notifications, profile
- Admin routes: (admin)/index, (admin)/tenants, (admin)/advertisers, (admin)/profile

## Key UI facts
- Login has quick-fill buttons: text "Admin", "Tenant 1", "Tenant 2"
- All Button components render text directly (tapOn by text works)
- Input component: label above TextInput — needs accessibilityLabel={label} on TextInput
- SearchBar: TextInput with no label — needs testID="search-input"
- Admin logout: icon-only Pressable — needs testID="admin-logout-btn"
- Property selector must be tapped before admin stats load
- Onboarding: 3 steps — property → unit → profile form
- CSVImporter: "Pick CSV" button text, uses expo-document-picker
- Segment controls: "All", "Tenant Offers", "Local Deals" in Promotions
- Community segments: "All", "Announcements", "Events"
- Admin tenant segments: "All", "Invited", "Active", "Inactive"
- Admin promotions segments: "Pending", "Approved", "Rejected"
