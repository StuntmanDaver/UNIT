## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-icon spacing |
| sm | 8px | Form field label-to-input gap, badge padding |
| md | 16px | Card internal padding, form section spacing |
| lg | 24px | Card-to-card vertical gap, page section padding |
| xl | 32px | Main content top padding from header |
| 2xl | 48px | Page vertical centering offset on auth screens |
| 3xl | 64px | Header height reservation (pt-32 on auth page = 128px, keep pattern) |

Exceptions:
- Touch target minimum for all interactive controls: 44px height (applied via `py-3` + natural line-height on buttons and inputs)
- Magic link send button: `py-6` (24px vertical padding) to match the existing login button height established in `LandlordLogin.jsx`
- Property switcher trigger: minimum 40px height, full-width on mobile

Source: Inferred from `LandlordLogin.jsx` patterns (`p-8`, `space-y-6`, `py-6`). Standard 8-point scale confirmed.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 16px (text-base) | 400 (normal) | 1.5 | Form helper text, paragraph content, notification copy |
| Label | 14px (text-sm) | 400 (normal) | 1.4 | Form field labels, table column headers, dropdown item labels |
| Heading | 20px (text-xl) | 700 (bold) | 1.2 | Card section headings, page sub-headings |
| Display | 28px (text-3xl) | 700 (bold) | 1.1 | Page title ("Landlord Access", "Audit Log"), property name in switcher |

Notes:
- Exactly 4 sizes, 2 weights: 400 (normal) for Body and Label; 700 (bold) for Heading and Display.
- The existing `LandlordLogin` uses `text-3xl font-bold` for the page title and `text-sm font-medium` for labels — this contract normalizes label weight to 400 to maintain the 2-weight rule.
- No italic styles in this phase.
- Monospace not used (magic link flow has no code-entry field).

Source: `LandlordLogin.jsx` line 71 (`text-3xl font-bold`), line 78 (`text-sm font-medium`). Confirmed default body sizing from shadcn new-york base.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#101B29` (`brand-navy`) / `hsl(var(--background))` in light mode | Auth page full-bleed background (`bg-gradient-to-br from-brand-navy via-brand-blue to-brand-navy`), landlord layout backgrounds |
| Secondary (30%) | `#1D263A` (`brand-blue`) / `hsl(var(--card))` | Cards (`bg-brand-navy/50 backdrop-blur-xl`), header bar (`bg-brand-navy/40 backdrop-blur-2xl`), dropdown panels |
| Accent (10%) | `#465A75` (`brand-slate`) / `#7C8DA7` (`brand-steel`) | Reserved for specific elements listed below |
| Destructive | `hsl(var(--destructive))` = `hsl(0 84.2% 60.2%)` | Destructive actions only (disable legacy code login, delete confirmations) |

**Accent reserved for:**
1. Primary CTA button gradient: `bg-gradient-to-r from-brand-slate to-brand-navy` (matches existing login button)
2. Property switcher active property indicator (left border or badge background)
3. Links and hover states on help/support text (`text-brand-slate-light hover:text-brand-steel`)
4. Focus ring on email input (`ring-brand-slate`)
5. Lock icon on auth page header illustration

**Accent NOT used for:** general body text, neutral UI chrome, card borders, timeline rows, table cells, pagination controls.

**Color notes:**
- Light mode (`hsl(var(--background))` = white) applies to any tenant-facing surfaces if any appear in this phase — keep landlord surfaces in dark navy treatment.
- Landlord pages stay in the dark navy brand palette regardless of system light/dark preference, matching the existing LandlordLogin pattern.
- Error inline states use `bg-red-500/10 border-red-500/30 text-red-400` (matches existing pattern in `LandlordLogin.jsx` line 96).

Source: `tailwind.config.js` brand tokens, `src/index.css` CSS variables, `LandlordLogin.jsx` class patterns, `project_brand_identity.md` memory.

## Interaction States

### Magic Link Login Form

| State | Behavior |
|-------|----------|
| Default | Email input empty, "Send Magic Link" button enabled |
| Typing | Real-time email format validation on blur only (not on keystroke) |
| Submitting | Button shows `<Loader2 className="animate-spin" />` + "Sending..." label, input disabled |
| Success | Form replaced with confirmation message: "Check your email" screen (see Copywriting) |
| Error (send failure) | Inline error banner below input (red-500/10 treatment, matching existing pattern) |
| Error (no account) | Same inline error banner — do NOT distinguish "no account found" from send failure (security) |

### Property Switcher Dropdown

| State | Behavior |
|-------|----------|
| Single property landlord | Switcher hidden entirely (D-09) |
| Multi-property landlord | Dropdown trigger shows active property name + `ChevronDown` icon |
| Open | DropdownMenu panel lists all properties, active property has checkmark or left-border accent |
| Switching | Trigger shows `Loader2` spinner while TanStack Query cache invalidates + new data loads |
| Switch complete | Dropdown closes, page data re-renders with new property, URL `?propertyId=X` updates |
| Switch error | Toast notification: "Failed to switch property. Try again." |

### LandlordGuard (Route Protection)

| State | Behavior |
|-------|----------|
| Loading auth | Full-page `Loader2` spinner centered, no flash of landlord content |
| Authenticated landlord | Renders children normally |
| Authenticated non-landlord | Redirect to `/Welcome` immediately |
| Unauthenticated | Redirect to `/LandlordLogin` immediately |
| No property context | Redirect to `/LandlordLogin` with toast: "Select a property to continue" |

### Audit Log Timeline

| State | Behavior |
|-------|----------|
| Loading | Skeleton rows (3 placeholder rows using `<Skeleton />`) |
| Empty | Empty state message (see Copywriting) |
| Populated | Scrollable list, newest entry first |
| Filtering | Filter controls at top; list updates in-place without page reload |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA — magic link | "Send Magic Link" |
| Primary CTA — property switch | "Switch Property" (DropdownMenu item label) |
| Email sent confirmation heading | "Check your email" |
| Email sent confirmation body | "We sent a sign-in link to {email}. Check your inbox and click the link to access your dashboard." |
| Magic link expired / invalid | "This sign-in link has expired or is invalid. Request a new link below." |
| Empty state — audit log heading | "No activity yet" |
| Empty state — audit log body | "Audit entries will appear here when financial records or request statuses are changed." |
| Error — magic link send failure | "Something went wrong sending your link. Check your email address and try again." |
| Error — property switch failure | "Failed to switch property. Please try again." |
| Error — auth required | "You need landlord access to view this page." |
| Error — wrong role | "Your account does not have landlord permissions." |
| Destructive — disable legacy code login | "Disable access code login" — confirmation: "This will remove the code-based login for all landlords. This cannot be undone. Type DISABLE to confirm." |
| Loading — auth check | (no text — spinner only, no "Loading..." label) |
| Property switcher label | "Switch property" (trigger tooltip for icon-only variant on mobile) |

**Tone:** Direct and informative. No exclamation marks. No filler phrases ("Oops!", "Uh oh!"). Error messages state the problem and the next action.

- Position: Right side of header nav, left of the notification bell
- Trigger: `Button variant="ghost"` showing `{propertyName} ChevronDown`
- Max width: 240px trigger, 280px dropdown panel
- Hidden when landlord has exactly 1 property (D-09)

- Full-width page layout matching existing Accounting page structure
