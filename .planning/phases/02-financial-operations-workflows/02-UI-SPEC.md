## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge internal padding |
| sm | 8px | Inline element spacing (`gap-2`), label-to-input gap (`mt-1`) |
| md | 16px | Default element spacing (`gap-4`, `space-y-4`) |
| lg | 24px | Section/card padding (`p-6`), horizontal page padding (`px-6`) |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks (not currently used but reserved) |
| 3xl | 64px | Page-level vertical rhythm (pt-24 header offset = 96px exception) |

Exceptions:
- Fixed header height: 64px (`h-16`) — matches xl but used as fixed structural constraint
- Main content top padding: 96px (`pt-24`) — accounts for fixed header + breathing room
- Header clearance is not a spacing token; it is a structural layout value

## Typography

| Role | Size | Weight | Line Height | Tailwind Class |
|------|------|--------|-------------|----------------|
| Body | 14px | 400 (normal) | 1.5 | `text-sm` |
| Label | 12px | 400 (normal) | 1.4 | `text-xs` |
| Heading | 20px | 700 (bold) | 1.2 | `text-xl font-bold` |
| Display | 24px | 700 (bold) | 1.2 | `text-2xl font-bold` |

Source: Inferred from `Accounting.jsx` and `LandlordRequests.jsx` — consistent usage of `text-sm` for body, `text-xl font-bold` for section headers, `text-2xl font-bold` for stat numbers, `text-xs` for metadata labels.

Additional weight allowed: `font-semibold` (600) for row-level primary text (e.g., business name in lists). Keep to 2 weights in practice: 400 for body/labels, 700 for headings/stats. Use `font-semibold` (600) only for list item primary text — not as a third general weight.

## Interaction Contracts

### Invoice Status Transitions (FIN-01)

Location: Invoice row in Accounting page → Invoices tab. Inline expand, not a modal.

Expand trigger: Click invoice row or chevron icon. Expands to show AuditLogTimeline (existing pattern) + InvoiceStatusActions.

InvoiceStatusActions renders only the NEXT ALLOWED transitions as buttons:
- `draft` → shows "Send Invoice" button (accent gradient)
- `sent` → shows "Mark as Paid" button (green-600 bg) + "Void" button (destructive variant)
- `overdue` → shows "Mark as Paid" button (green-600 bg) + "Void" button (destructive variant)
- `paid` → no action buttons (terminal state); show read-only "Paid" badge
- `void` → no action buttons (terminal state); show read-only "Void" badge

"Send Invoice" sends a transactional email (COMM-01). Show a loading spinner on the button during the Edge Function call. On success: invalidate invoices query + show sonner toast "Invoice sent to [email]".

"Void" uses AlertDialog confirmation (see Copywriting Contract below).

### SLA Escalation (REQ-05)

Location: LandlordRequests page — request list rows.

- SlaDeadlineBadge renders: "Due in 2d" (amber-400 on dark bg) when deadline is in the future; "Overdue — 1d" (red-400 on dark bg) when past deadline.
- Escalated requests: `escalated` boolean = true → render red "Escalated" Badge at the start of the row title + float to top of list (sort escalated=true first in query result).
- SLA deadline and escalation state are shown to landlords only. They must not appear on `Recommendations.jsx` (tenant view).

### Request Assignment (REQ-01)

Location: LandlordRequests page — request detail expand panel (same expand-row pattern as invoice audit).

AssigneeField renders as:
- Display mode: Assignee name/email as `text-sm text-zinc-300`. If unassigned: "Unassigned" in `text-zinc-500`.
- Edit mode: Input field + "Assign" button (accent gradient). Triggered by clicking a pencil icon (`Pencil` from Lucide, 14px).
- On save: useMutation to update `assigned_to`, invalidate recommendations query, show toast "Request assigned to [name]".

### Export Controls (FIN-04, FIN-05)

Location: Accounting page — tab-level toolbar area (right side of each tab header).

ExportControls renders two buttons:
1. "Export CSV" — `variant="outline"`, size sm. Icon: `Download` (Lucide, 14px). Triggers vanilla JS CSV download. No loading state needed (synchronous).
2. "Export PDF" — `variant="outline"`, size sm. Icon: `FileDown` (Lucide, 14px). Shows `Loader2` spinner while jsPDF generates. Disabled during generation.

PDF branded layout (D-17):
- Header: UNIT logo (`src/components/UnitLogo.jsx`) + property name, right-aligned
- Header accent bar: 4px solid `#465A75` (brand-slate) below header row
- Body font: 10pt, black
- Table header row: `#1D263A` fill (brand-blue), white text
- Table row alternating: white and `#F8F8F8`
- Footer: "Generated by UNIT — [date]" in 8pt gray

### Tenant Invoice Page (D-04)

Route: `/my-invoices` or via tenant nav entry (Claude's discretion on exact route placement).

Layout: Standard page layout matching Recommendations.jsx pattern (not the dark landlord layout). Use the tenant-side light/default theme.

Page shows: TenantInvoiceCard list filtered by the authenticated user's business. Each card shows invoice number, amount, due date, and status badge. No action buttons in Phase 2 (Phase 4 adds "Pay Invoice").

Empty state: See Copywriting Contract.
